关于 Octopus torum 相关信息的面试整理

Octopus Network

  • Octopus Network 要解决什么问题 ？

应用链的安全自举需要耗费大量资金的问题，不灵活且成本高。

PoS 链的安全性依赖于 Saking 资产的价值。要加密社区认可通证有价值，通常需要几百万美金和很长时间，然后大家才会协作 POS，构建应用链的安全性。

应用链的跨链桥开发难的问题。

跨链桥不好维护，应用链团队自己开发部署跨

  • Octopus Network 解决方案和其它的有什么不同 ？ 有什么优势 ？

Staking 的解锁期是 3 周，Delegtion 解锁期是 1 周。

比参与插槽拍卖众贷风险低，Polkadot 的众贷锁 2 两年，Kusama 的锁 1 年。平行链的业务如果发展得不好，币价归零了，参与众贷的人也只能等 1、2 年才能拿回本金。而且 DOT 和 KSM 都是不断增发的，你的币被锁住就会被稀释，就是在向系统缴纳铸币税。OCT 通证是不增发的，你不作应用链 Staking 也不会被稀释，做 Staking 的收益扣除 IT 成本，就是纯利。

章鱼网络的设计目标就是大幅降低应用链的部署和运行成本，让几个人的小型创业团队也能开展应用链创新，从而为实现 Web3.0 创造条件。

  • **从用户角度说说我的看法 **

流程上来说我的App链加入章鱼网络后先要通过审计，然后经过OCT持有者投票后进入质押，打到最低质押限制后才能启动。

从官网看目前有一个应用链处于启动中，质押数74w OCT ，审计费用为1000 OCT ，OCT 持有者成为验证人至少需要1w otc， 最低安全质押OCT限制为200w美金， 目前数据量暂不能支撑 大幅度降低应用链成本的结论。

torum

  • torum 要解决什么问题 ？

以区块链为底层技术的社交平台，并且专门是为加密货币爱好者而建

Torum增加了用户行为奖励机制，并内置了一项加密货币钱包功能，用户可通过完成系统所分发的游戏化任务来获取平台代币XTM作为奖励。随着更多用户和项目的加入，Torum平台将逐渐成为更加繁荣的社交生态系统。

  • torum 解决方案和其它的有什么不同 ？ 有什么优势 ？

相较于完全去中心化的区块链社交平台（如Steemit、HIVE等），Torum采用的是去中心化与中心化的混合模型（Hybrid Decentralized Model），将社交平台以中心化的架构搭建，并通过连接置入Web3 与币安智能链中的智能合约连接，同时让用户能够将自身的浏览器插件钱包如MetaMask与社交平台账号绑定，从而访问平台中各类去中心化服务。用户可以方便地使用DeFi 质押、流动性挖矿、NFT交易等服务，并可随时分享到其他平台上进行展示、交流。

而相较于其他区块链社交平台，Torum则摆脱了一般Dapp使用难度大的“刻板印象”，其UI用户界面更加友好，可以保证任何新成员轻松地熟悉起来，只需10分钟便可掌握Torum的各种功能。此外，Torum还融合了UI / UX 等元素，在功能选择、视觉方面也更加强大。

  • **从用户角度说说我的看法 **

初体验注册发现使用中国手机号会停留在 等待界面不能完成注册，用其它国家手机号注册后得以登陆。

从用户体验上来说存在首屏加载时间5s以上，有DeFi、NFT 等多个功能用户指引不够完善，UI界面略显杂乱， Icon 语义不明等问题。

My Portfolio

奥德赛阅读 （亚月科技 / 2016 - 2019）

  • 初创团队的第一个项目，也是自己第一个自己独立开发的前端项目。主要服务于英语的进阶者，提供阅读，听力，口语和写作四个类别的服务，目的是解决英语学习者在学习中原著阅读不顺畅，英语内容写作不能得到很好的反馈等问题。
  • 负责 原著阅读，写作互评，口语互评四个工程的前端开发，及其后台的内容管理，用户管理，产品管理，学生评分模块的开发。
  • 用户前端采用Vue + vuex 的技术栈，不同工程采用不同的 SPA工程，部署在不同的域名，使用单点登陆。不同前端工程共用一套封装后的 mintUI 进行组件复用。 组件库采用私有 npm 包管理。
  • vue成本低，开发速度快，项目如期上线，由于前端团队人数并不多，并没有遇到难以维护的问题。我在应用中也强化了对前端技术的理解。

网络摄像头DH-IPC / 大华通用组件库 （大华科技 / 2019 - 2021）

  • 前端优化项目，通用组件以及脚手架，目的是解决大华技术栈历史包袱过重，各业务线组件难以复用，UI设计不统一，以及导致的招聘问题。
  • 负责 部分通用组件的开发以及维护，通用库第一个业务应用的开发以及维护。
  • 用户前端采用React + Anted 的技术栈，通用库 Anted 封装，npm 私有库管理，storybook作为文档维护工具，采用 Jest 进行组件测试。开发了时间计划表组件解决老组件的逻辑难以复用和用户体验不佳的问题，设计了通用的 grid 布局解决多语言长度页面显示不一致的问题，减少了定制化样式的工作量。
  • 通用库的第一个应用落地后，逐渐开始被其它业务线使用，虽然也出现了因为通用开发维护周期过长，为避免项目延期测试用例不完善的问题，但也确实解决了UI不同一和部分组件复用的问题。
